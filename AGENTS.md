# portfolio Agent Guide

Portfolio project.

## Scope

This folder is a `career` workspace/project area. Treat it as its own work package unless a nearer `AGENTS.md` says otherwise.

## Rules

- Preserve polished PDFs, resumes, portfolio exports, and original source material before major edits.
- Keep tone direct and professional.
- Tailor outputs to the target role/company when job context exists.
- Check this folder's `TODO.md` before starting work.
- Keep edits scoped to this folder unless the user asks for cross-project changes.
- Preserve source inputs and create generated outputs separately when practical.
- Update `README.md`, `TODO.md when behavior or workflow changes.
- Before final response, run the smallest useful verification and report git status for the active repo or workspace root.

## Verification

- Prefer project-specific tests or sample runs when available.
- If runtime verification is not available, do a read-only review of changed files and state what remains unverified.
- Report `git status --short --branch` from this folder if it is a repo; otherwise report workspace-root status.

## TODO lookup rule

- Check this folder's `TODO.md` first.
- If it has no relevant active item, check the nearest parent `TODO.md`.
- Do not create TODO files in raw data, generated output, cache, fixture, or build folders.
